分布式版本管理系统（Git）和集中式版本管理系统（SVN）
Git和SVN最大的不同：
说法一：使用Git时，在没有网络的的情况下，可以将最新版本提交至本地，在本地做版本管理所有处理。比如提交一个版本1，提交版本2，提交版本3，然后恢复版本1，开一个分支，切换分支，修改代码，合并分支。Git在无网络情况下可以本地提交，他把提交版本与推送服务器这两个概念分开了。而svn将这个融为一体，提交即是推送服务器。
说法二：这两个工具主要的区别在于历史版本维护的位置，Git本地仓库包含代码库和历史库，在本地的环境开发就可以记录历史而SVN的历史库存在中央仓库，每次对比与提交代码都必须连接到中央仓库才能进行

	- ==-----
 git add将文件修改添加到暂存区
 git commit提交更改就是把暂存区所有内容提交到当前分支
 git diff #是工作区（work dir）和暂存区比较（stage）的比较
 git diff --cached #是暂存区（stage）和分支（master）的比较
 git diff HEAD -- readme.txt 工作区和分支的对比

 


 git diff <file>和git diff HEAD -- <file>不同，前者会单独区分add前后的变化，是在工作区和缓存区之间的比较。而后者则是在工作区和最新分支之间的比较（应该是上一次commit后的分支）

  git status是比工作区，缓存区，分支的所有变化，并且给出每个阶段的状态。
